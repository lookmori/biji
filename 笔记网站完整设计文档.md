## 1. 文档概述

### 1.1 项目背景

市面上主流笔记工具存在三大痛点：

1. **图形编辑干扰写作** — 打开思维导图/流程图会全屏挤占笔记编辑区域，无法边写正文边梳理逻辑
2. **画布功能薄弱** — 原生画布能力不足，无法自由手绘批注
3. **图文排版割裂** — 图形内容和正文笔记切换繁琐

本项目基于 Next.js 16 App Router 搭建轻量化沉浸式笔记工作台：

- 沿用左侧文件目录 + 右侧主编辑区经典双栏布局
- 全面强化 Canvas 原生绘图能力，将流程图、思维导图、手绘批注全部内置悬浮画布面板
- 图形编辑完成后自动转为图片嵌入正文
- 面板支持最小化悬浮、鼠标移入自动展开，彻底解决图形编辑与正文写作互相干扰

### 1.2 核心设计目标

| 目标 | 说明 |
|---|---|
| 固定双栏布局 | 左侧树形文件目录永久固定，右侧为笔记正文主编辑区 |
| 悬浮图形面板 | 流程图、思维导图独立悬浮画布面板，支持一键最小化至悬浮球 |
| 图形图片化嵌入 | 思维导图、流程图编辑完成后一键生成高清图片，直接插入富文本笔记正文 |
| 全能 Canvas 能力 | 自由手绘、基础图形、文字批注、橡皮擦、撤销重做、图层管理 |
| 🆕 富媒体支持 | 图片、视频嵌入、音频、文件附件全类型内容 |
| 🆕 首页动画 | GSAP 动画 + undraw 插画打造品牌首页 |
| 无感自动保存 | 正文、画布数据、面板状态全部 2 秒防抖自动云端保存 |
| 统一视觉规范 | 完整复用参考文档色彩、字体、间距、圆角、阴影全套 UI 规范 |

### 1.3 受众人群

学生笔记梳理、产品经理需求绘图、程序员架构流程图绘制、职场人士会议纪要撰写、内容创作者富媒体笔记整理。

## 2. 整体设计原则

### 2.1 内容优先

- 所有悬浮面板、画布控件均为写作服务，默认不抢占正文编辑区视觉焦点
- 最小化状态下仅保留极小悬浮图标，保证正文写作无视觉干扰
- 首页之外的页面，一屏内优先显示当前笔记内容和编辑工具

### 2.2 克制视觉

- 无渐变背景、无高饱和装饰、无冗余卡片堆叠
- 所有控件阴影、圆角、色彩严格遵循统一规范
- 界面干净舒适，长时间写作无视觉疲劳
- 🆕 首页允许适度的品牌动画和插画表达，但克制在不喧宾夺主的程度

### 2.3 秩序稳定

- 面板展开/收起、画布缩放、图片插入均不会导致正文布局抖动
- 所有画布元素基于固定网格绘制，对齐精准，排版稳定
- 控件、卡片和画布必须具有稳定尺寸

### 2.4 双态交互

- 图形面板分两种状态：展开态（完整画布编辑）、悬浮态（仅悬浮球，hover 展开）
- 用户可自由锁定展开/悬浮状态，适配不同写作习惯

### 2.5 🆕 品牌表达

- 首页作为品牌入口，承担价值传达和功能导航双重任务
- 使用 GSAP 制作流畅、有质感的入场动画
- 使用 undraw 插画作为视觉辅助，保持风格统一

## 4. 全局页面布局设计

```

### 4.2 顶栏规范（对齐参考文档）

- 高度：`72px`
- 固定在视口顶部
- 背景：`rgba(247,248,243,.92)`
- 使用 `backdrop-filter: blur(12px)` 的玻璃模糊效果
- 下边框：`#D8E0D9`
- 层级：`z-index: 30`

内容：
- 左侧：品牌图标 + "BIJI" 中文名 + 英文副标

- 右侧：全局搜索、打印按钮、用户头像
- 移动端左侧增加 40×40px 菜单按钮

### 4.3 侧栏规范

- 桌面宽：`232px`
- 桌面断点：`md`（约 768px）
- 背景：`rgba(250,251,247,.72)`
- 右边框：`#DCE3DD`
- 内边距：`20px`
- 导航项：12px 上下内边距，圆角 12px
- 激活项：深绿底 + 白字 + 中等级阴影
- 禁用项：`opacity: .55`

移动端：
- 左侧抽屉，宽 `285px`
- 遮罩：`rgba(0,0,0,.3)`
- 抽屉层级：`z-index: 50`

### 4.4 悬浮画布面板布局（右下角固定）

#### 4.4.1 布局边距硬性规范

| 参数 | 数值 |
|---|---|
| 悬浮球距页面右边缘 | 20px |
| 悬浮球距页面底部 | 24px |
| 两个悬浮球垂直间距 | 16px |
| 展开面板距浏览器右边缘 | 20px |
| 展开面板距浏览器底部 | 24px |
| 悬浮层级 | `z-index: 60` |

#### 4.4.2 最小化悬浮态

- 两个 40×40px 圆形悬浮球，垂直堆叠
- 悬浮球底色：`green-dark (#173E31)`，图标白色，小圆角 8px
- 鼠标悬浮 300ms 延迟再展开，避免误触
- 鼠标移出面板可视区域 400ms 后自动缩回
- 支持锁定面板展开状态

#### 4.4.3 完全展开态

- 面板尺寸：固定宽 380px，高 70vh
- 展开方向：从右下角向左 + 向上平滑展开
- 过渡动画：460ms ease

## 5. 首页设计（GSAP + undraw 插画）🆕

### 5.1 首页定位

首页是 BIJI 笔记工作台的品牌入口页面，承担以下职责：

1. **价值传达** — 用一句话说清楚 BIJI 是什么、为什么好用
2. **功能展示** — 展示核心功能（Canvas 画图、富媒体笔记、悬浮面板）
3. **行动引导** — 引导用户注册/登录/开始使用
4. **品牌记忆** — 通过 GSAP 动画和 undraw 插画留下视觉印象

### 5.2 首页整体结构

```

│   🎯 Hero 区（深色背景 #0F1D18，最小 85vh）              │
│   ┌─────────────────────────────────────────────┐    │
│   │  左侧文字（GSAP 入场动画）                      │    │
│   │  - 品牌名 + 主标题（逐字浮现）                   │    │
│   │  - 副标题描述（从下升起）                       │    │
│   │  - CTA 按钮组（延迟出现）                       │    │
│   │                                               │    │
│   │  右侧插画（undraw 插画 + GSAP 浮动动画）         │    │
│   │  - 主视觉插画（笔记/写作主题）                    │    │

#### 5.3.4 GSAP 动画设计

##### 动画时间线

| 阶段 | 时间 | 动画内容 |
|---|---|---|
| 1 | 0ms | 页面加载完成 |
| 2 | 0–200ms | Hero 背景淡入（opacity 0→1） |
| 3 | 200–600ms | 主标题文字逐字浮现（stagger 效果） |
| 4 | 400–700ms | 副标题从下升起（translateY 20px→0） |
| 5 | 600–900ms | CTA 按钮淡入 + 轻微上移 |
| 6 | 300–1200ms | 右侧插画从右侧滑入 + 持续浮动 |
| 7 | 持续 | 插画呼吸浮动动画（yoyo 循环） |

##### 具体 GSAP 实现方案

```javascript
// 首页 GSAP 动画时间线
import { useEffect, useRef } from 'react';
import gsap from 'gsap';
import { ScrollTrigger } from 'gsap/ScrollTrigger';

gsap.registerPlugin(ScrollTrigger);

// Hero 入场动画
const heroTimeline = gsap.timeline({ defaults: { ease: 'power3.out' } });

heroTimeline
  // 背景淡入
  .fromTo('.hero-bg', 
    { opacity: 0 }, 
    { opacity: 1, duration: 0.6 }
  )
  // 标题逐字浮现（使用 SplitText 或手动 stagger）
  .fromTo('.hero-title .char', 
    { opacity: 0, y: 20 }, 
    { opacity: 1, y: 0, stagger: 0.04, duration: 0.5 },
    '-=0.3'
  )
  // 副标题升起
  .fromTo('.hero-subtitle', 
    { opacity: 0, y: 24 }, 
    { opacity: 1, y: 0, duration: 0.46 },
    '-=0.2'
  )
  // CTA 按钮出现
  .fromTo('.hero-cta', 
    { opacity: 0, y: 12 }, 
    { opacity: 1, y: 0, duration: 0.4 },
    '-=0.2'
  )
  // 插画滑入
  .fromTo('.hero-illustration', 
    { opacity: 0, x: 60 }, 
    { opacity: 1, x: 0, duration: 0.8, ease: 'power2.out' },
    '-=0.6'
  );

// 插画持续浮动动画
gsap.to('.hero-illustration', {
  y: -12,
  duration: 3,
  repeat: -1,
  yoyo: true,
  ease: 'sine.inOut',
  delay: 1.2
});

// 功能卡片滚动触发动画
const cards = gsap.utils.toArray('.feature-card');
cards.forEach((card, i) => {
  gsap.fromTo(card, 
    { opacity: 0, y: 40 },
    {
      opacity: 1, y: 0,
      duration: 0.6,
      scrollTrigger: {
        trigger: card,
        start: 'top bottom-=80px',
        toggleActions: 'play none none none',
      },
      delay: i * 0.15
    }
  );
});
```

##### 动画设计原则

- 所有入场动画在 `prefers-reduced-motion: no-preference` 下执行
- 用户开启减少动效后，直接显示最终状态，不播放动画
- 总动画时长控制在 1.5 秒以内，不拖沓
- 插画浮动动画幅度不超过 12px，避免分散注意力
- 页面滚动后的功能卡片动画使用 ScrollTrigger，仅触发一次

## 12. 动效交互规范

### 12.1 GSAP 动画（首页专用）🆕

| 动画 | 时长 | 缓动 | 说明 |
|---|---|---|---|
| Hero 标题逐字浮现 | 500ms | `power3.out` | stagger 0.04s/字 |
| 副标题升起 | 460ms | `ease` | translateY: 24px → 0 |
| CTA 按钮淡入 | 400ms | `ease` | translateY: 12px → 0 |
| 插画滑入 | 800ms | `power2.out` | translateX: 60px → 0 |
| 插画浮动 | 3000ms | `sine.inOut` | yoyo 循环 ±12px |
| 功能卡片滚动入 | 600ms | `ease` | ScrollTrigger 触发 |

### 12.2 通用动效（对齐参考文档）

- 页面进入 `.rise` 动画：460ms ease，opacity 0→1，translateY 8px→0
- 卡片 hover：上移 2–4px，阴影加深
- 主按钮 hover：上移 2px
- 面板展开/收起：460ms ease
- 所有动效置于 `prefers-reduced-motion: no-preference` 中

### 12.3 悬浮面板动效

- 展开：透明度 0→1，宽度 0→380px，460ms ease
- 收起：透明度 1→0，宽度 380px→0，460ms ease
- 🆕 悬浮球 hover：放大 1.06 倍，150ms ease

## 14. 风险点与解决方案

| 风险点 | 解决方案 |
|---|---|
| 悬浮面板 hover 展开干扰写作 | 300ms 延迟触发；支持锁定面板状态 |
| Canvas 数据量大，自动保存卡顿 | 画布数据 3 秒防抖，正文 2 秒防抖，分开存储 |
| 图形图片与原始画布数据不同步 | 每次保存自动覆盖预览图，打开笔记拉取最新数据 |
| 🆕 视频文件过大，上传缓慢 | 前端分片上传 + Vercel Blob 大文件支持；超过 100MB 提示压缩 |
| 🆕 GSAP 动画在低端设备卡顿 | 检测设备性能，低端设备简化动画（跳过 stagger，减少浮动幅度） |
| 🆕 暗色模式 Canvas 对比度不足 | 暗色模式下自动切换 Canvas 背景色和线条色 |
| 移动端悬浮面板操作不便 | 移动端隐藏悬浮面板，改为底部 Tab 唤起画布，全屏编辑 |
| undraw 插画加载慢 | 使用 SVG 内联而非外部加载，利用 Next.js 静态优化 |
